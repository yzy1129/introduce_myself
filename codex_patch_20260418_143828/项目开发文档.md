# 项目开发文档

## 1. 项目定位

本项目是杨子煜的沉浸式个人数字空间作品集，核心目标不是“罗列我做过什么”，而是通过空间、动效、交互和项目证据，让访问者逐步建立对能力结构与个人风格的认知。

用户路径：

进入宇宙 -> 接近核心 -> 理解个体 -> 探索能力 -> 验证项目 -> 建立连接

## 3. 设计稿与实现对照

### 3.1 已对齐

- 能量场替代普通鼠标
- Reveal 作为统一信息展示机制
- 粒子宇宙作为全局背景
- 滚动驱动动画
- 3D About 形象
- Skills 旋转轨道与拖拽
- Projects 全屏沉浸查看
- 声音反馈系统

### 3.2 当前采用的工程化落地方式

- Hero 的“点击中心进入下一章”已做成引力核心按钮，而不是额外相机系统
- 项目沉浸模式采用全屏卷宗 Overlay + 独立详情页并存
- 声音系统当前使用 Web Audio 合成反馈音，不依赖外部音频文件
- favicon 采用根据主视觉提炼的 SVG 图形，保证小尺寸下仍可识别

### 3.3 暂未继续扩展的方向

这些不是缺失，而是后续可升级项：

- 更强的 Shader 折射与体积光
- 更复杂的粒子文字生成
- 明暗主题切换
- 真实音效素材替换当前合成音
- 项目详情中的视频、架构图、演示录屏

## 4. 内容维护规则

主要内容统一维护在：

```text
src/content/siteContent.ts
```

可直接修改：

- 首页标题与副标题
- About 文案
- 技能节点
- 项目条目
- Timeline
- 联系方式与终端命令

## 5. 静态资源规则

静态资源统一放在：

```text
public/
```

构建时会复制到 `dist/`。

如果后续要手动补真实音频资源，建议使用：

```text
public/audio/
```

建议素材类型：

- `ambient-space.*`
  持续氛围音，弱节奏，低频深空感
- `hover-soft.*`
  轻、短、空灵的悬停提示
- `click-pulse.*`
  短促点击脉冲
- `route-warp.*`
  页面切换推进音

## 6. 本地开发

```bash
npm.cmd run dev
npm.cmd run lint
npm.cmd run build
```

如需静态预览：

```bash
npm.cmd run preview
```
